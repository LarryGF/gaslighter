# Gaslighter Plugin

Claude Code plugin that uses a Stop hook to nudge Claude into verifying requirement completeness.

## How It Works

Each time Claude tries to finish a response, the Stop hook fires a short psychologically-effective nudge asking it to re-read the original request. Anti-loop guard: capped nudges per session (see Modes below). First nudge forces re-examination; subsequent nudges give an escape hatch ‚Äî if the model's last turn (read from `transcript_path`) already declares "100% certain/confident/sure", the hook stops nudging immediately instead of counting up to the cap regardless of what was said.

## Modes

Mode and nudge cap are persisted to `${CLAUDE_PLUGIN_DATA}/config.json` (`{ "mode": "lite", "maxNudges": 3 }`, `maxNudges` optional). Set via the `gaslighter:config` skill. Precedence for each setting independently: env var > persisted `config.json` > mode default.

Both delivery modes deliver via stdout + exit 0 (per the [Stop Hook Reference](https://code.claude.com/docs/en/hooks.md#stop-hook-reference), the JSON decision protocol only works this way):
- `lite` (default) ‚Äî `hookSpecificOutput.additionalContext`, non-blocking soft nudge, default cap 3
- `full` ‚Äî `decision: "block"`, hard block, default cap unlimited
- `off` ‚Äî disabled, cap always 0

`GASLIGHTER_MODE` overrides the persisted mode; `GASLIGHTER_MAX_NUDGES` overrides the persisted/default cap (`infinite`/`unlimited`/`-1` for no cap).

## Structure

- `.claude-plugin/plugin.json` ‚Äî plugin manifest
- `.claude-plugin/marketplace.json` ‚Äî self-hosted marketplace listing (public install path)
- `hooks/` ‚Äî hook scripts (Node.js, CommonJS)
  - `gaslighter-hooks.json` ‚Äî hook registration (Stop only)
  - `gaslighter-nudge.js` ‚Äî Stop hook: mode/config check, anti-loop guard, emit nudge
  - `gaslighter-config-cli.js` ‚Äî CLI wrapper for reading/writing persisted config.json
  - `package.json` ‚Äî CommonJS marker
- `agents/judge-agent.md` ‚Äî judge sub-agent definition (rubrics, calibration)
- `skills/gaslighter/SKILL.md` ‚Äî orchestrator skill (routes to eval/judge/config)
- `skills/judge/SKILL.md` ‚Äî eval judging orchestrator (fans out to judge-agent)
- `skills/config/SKILL.md` ‚Äî persists mode/nudge cap to config.json
- `evals/` ‚Äî benchmark suite
- `assets/` ‚Äî README benchmark chart SVGs, auto-generated by `evals/render_findings.py` (also via `--chart`)
- `tests/` ‚Äî unit tests

## Running tests

```bash
node tests/test-nudge-decision.js
```

## Key conventions

- Hook paths use `${CLAUDE_PLUGIN_ROOT}`, never `CLAUDE_SKILL_DIR`
- Session state stored in `${CLAUDE_PLUGIN_DATA}/state-{session_id}.json`
- Config (mode/maxNudges) persisted in `${CLAUDE_PLUGIN_DATA}/config.json`, overridable via `GASLIGHTER_MODE` / `GASLIGHTER_MAX_NUDGES` env vars
- All hooks have `commandWindows` equivalents
