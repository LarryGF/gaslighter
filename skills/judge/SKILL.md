# Gaslighter Judge

Judge a completed `runs/<timestamp>` directory.

Resolve `<plugin-root>` from this loaded skill's location. Run every command below from `<plugin-root>`; do not assume the caller's cwd is the gaslighter checkout.

## Prepare

Run:

```sh
cd "<plugin-root>" && python3 evals/judge.py --prep evals/runs/<timestamp>
```

The command writes one `judge_input/<task>.json` file per task and prints a manifest containing each task's path and workspace count.

## Fan out

Use the host's parallel task/subagent facility: launch one read-only evaluator per task group in a single batch.

Each evaluator must:

1. Read its entire `judge_input/<task>.json`.
2. Score every workspace against its task prompt.
3. Use `complete_rate` only as a calibration anchor.
4. Return **only** this JSON shape, with one entry per input workspace:

```json
{
  "scores": [
    {
      "task": "<task>",
      "arm": "<arm>",
      "model": "<model>",
      "workspace": "<copied verbatim>",
      "completeness": 0,
      "missing": "<most important missing piece or none>",
      "overcorrection": 0,
      "cite": "<most unnecessary construct or none>"
    }
  ]
}
```

Scores use completeness 0–3 and overcorrection 0–3. Never sample, duplicate, fabricate, or alter `workspace`.

## Merge

1. Extract each evaluator's outer JSON object; retry only failed evaluators.
2. Write each object to `evals/runs/<timestamp>/judge_input/<task>.scores.json`.
3. Run:

   ```sh
cd "<plugin-root>" && python3 evals/judge.py --merge evals/runs/<timestamp>
   ```

4. If merge reports missing or duplicate workspaces, rerun only that task's evaluator.
5. On success, display the per-task/per-arm completeness and overcorrection table.

## Render

Run:

```sh
cd "<plugin-root>" && python3 evals/render_findings.py evals/runs/<timestamp>
```

Read its delta table. Update only the interpretive `## Key findings` section in `docs/eval-findings.md`; automated sentinel regions and charts are generated by the script.
